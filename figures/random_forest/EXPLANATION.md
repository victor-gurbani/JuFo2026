# Understanding the Random Forest Artifacts

This directory contains the visual and textual artifacts generated by the interpretable Random Forest classifier (`src/classify_rf.py`). Because Random Forests are traditionally "black box" models, these artifacts are generated to provide full mathematical transparency into how the model makes its stylistic decisions.

## 1. Decision Trees (`rf_sample_tree.png` & `all_trees/`)

A Random Forest is an ensemble (committee) of many individual decision trees. When classifying a new piece, all trees vote, and the majority wins.

- **`rf_sample_tree.png`**: The single most accurate tree in the forest.
- **`all_trees/`**: Contains every individual tree in the forest for complete auditing.

### How to Read a Tree Node
Each box (node) in the tree visualizes a step in the decision-making process:
* **Feature Condition (e.g., `micro_rhythmic_density <= 0.45`)**: The rule used to split the data. If true, follow the left branch (usually representing "True"); if false, follow the right branch.
* **`gini` (Gini Impurity)**: A mathematical measure of "chaos" or "mixture" in that node. 
  * A Gini of `0.0` means the node is perfectly pure (e.g., it contains *only* Chopin pieces). 
  * A higher Gini (max 0.75 for 4 equally distributed classes) means the node contains a mixed bag of composers. The algorithm's goal at every split is to reduce Gini as much as possible.
* **`samples`**: The number of music pieces from the training data that reached this specific node.
* **`value`**: An array showing how the samples are distributed across the chronological classes: `[Bach, Mozart, Chopin, Debussy]`. 
  * *Example: `[0, 5, 0, 0]` means 0 Bach, 5 Mozart, 0 Chopin, 0 Debussy.*
* **`class`**: The model's prediction if the decision path ends at this node (the majority vote of the `value` array).

## 2. SHAP Summary Plot (`rf_shap_summary.png`)

While a single tree shows *micro-logic*, SHAP (SHapley Additive exPlanations) visualizes the *macro-logic* of the entire forest combined. It uses game theory to calculate the exact contribution of each feature.

* **Y-Axis (Top to Bottom)**: Features are ranked by global importance. The top feature is the most defining characteristic across the entire dataset.
* **Dots & Colors**: Each dot represents one piece of music. 
  * **Red dot**: The piece had a *HIGH* value for that feature.
  * **Blue dot**: The piece had a *LOW* value for that feature.
* **X-Axis (Left to Right)**: Shows how the feature shifted the model's prediction. 
  * *Example*: If you look at the "Bach" section, and the feature `polyphonic_density` has a cluster of **Red** dots pushed far to the **Right**, it means: "When a piece has high polyphonic density, the model strongly leans toward predicting Bach."

## 3. Confusion Matrix (`confusion_matrix.png`)

This is a 4x4 grid showing exactly where the model got things right and where it got confused, sorted chronologically (Bach -> Mozart -> Chopin -> Debussy).

* **Diagonal (Top-Left to Bottom-Right)**: True Positives. Pieces the model predicted correctly. You want these numbers to be as high as possible.
* **Off-Diagonal**: Errors. For example, if row is "Mozart" and column is "Bach", the number indicates how many actual Mozart pieces the model mistakenly thought were Bach.

## 4. Precision vs. Recall Scatterplot (`classification_scatter.png`)

This plot evaluates the model's reliability for each specific composer.

* **Recall (X-axis / Completeness)**: "Out of all the actual Chopin pieces, how many did the model find?"
  * Low recall means the model is "missing" pieces by that composer.
* **Precision (Y-axis / Exactness)**: "When the model *claims* a piece is Chopin, how often is it actually correct?"
  * Low precision means the model is "crying wolf" and guessing that composer too often.
* **Ideal Position**: The top-right corner `(1.0, 1.0)`.

---
*Note: These artifacts are generated automatically using Cost-Complexity Pruning to ensure the trees remain shallow and human-readable, sacrificing a tiny fraction of accuracy for a massive gain in interpretability.*
